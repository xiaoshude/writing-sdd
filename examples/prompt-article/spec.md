## 核心定义

### 一句话主题
用 SDD（规范驱动开发）的思想，让你的编程原则被 AI 一致执行，同时沉淀可复用的 prompt 资产。

### 核心论点
> 大多数人用 AI 编程的痛苦，不是 AI 不够强，而是你的意图没有被结构化地表达和积累。

### 独特视角
- 借鉴 SDD 中「规范优先」的理念，将其从代码生成迁移到 **prompt 工程**
- 不是教你怎么用 SDD 写代码，而是教你怎么用 SDD 的思想 **驯服 AI**

### 目标读者
- 已经在用 Cursor / Copilot / Claude 写代码的开发者
- 有自己的编程原则（如单向数据流、不可变数据），但苦于每次都要重复教 AI
- 想把「和 AI 协作的经验」沉淀下来，而不是每次从零开始

### 读者收获
1. 理解为什么你的 prompt 不被 AI「记住」——缺乏结构化表达
2. 学会用 SDD 思想构建自己的 prompt 体系
3. 带走一套可直接复用的 prompt 模板（如 AGENTS.md）

### 写作模式
- **主模式**: `lessons-learned`（经验教训型）
- **辅助模式**: `thoughts-on-trends`（行业趋势型）

## 文章大纲（草案）

### 1. 开头：共鸣痛点
- 你有没有这种感觉：每次和 AI 协作，都要重新「教育」它？
- 你的编程原则写在 cursor rules 里，但 AI 真的每次都遵守吗？

### 2. 问题本质：意图没有被结构化
- prompt 是「意图的表达」
- 碎片化的 prompt → 碎片化的执行
- 案例：数据不可变原则被 AI 忽略

### 3. 解法：借鉴 SDD 的核心思想
- SDD 的核心：规范 > 代码
- 迁移到 prompt 工程：prompt 规范 > 单次对话
- 关键机制：
  - 模板约束 LLM 输出
  - `[NEEDS CLARIFICATION]` 标记不确定性
  - 阶段性关卡验证

### 4. 实操：如何构建你的 prompt 体系
- 第一步：提炼你的编程原则
- 第二步：结构化表达（用 AGENTS.md 模板）
- 第三步：建立沉淀机制

### 5. 案例：一个优秀的 prompt 长什么样
- 解析 AGENTS.md 的结构
- 为什么它有效

### 6. 结尾：少吃低级的苦
- 有些苦不值得吃：重复教 AI、每次从零开始
- 把精力留给更高级的挑战
