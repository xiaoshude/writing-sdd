# /writing.plan

生成或优化文章大纲。

## 用法

```
/writing.plan [文章目录]
```

## 前置条件

- `spec.md` 核心定义已填写（至少填写一句话主题和核心论点）

## 执行步骤

1. **读取规格**
   - 读取 `spec.md` 的核心定义
   - 读取 `素材/` 目录下的原始材料

2. **分析写作模式**
   根据 `spec.md` 中指定的写作模式确定大纲结构：
   
   | 模式 | 典型结构 |
   |------|----------|
   | lessons-learned | 痛点 → 踩坑过程 → 教训 → 建议 |
   | bug-hunt | 现象 → 排查过程 → 根因 → 修复 |
   | how-we-built-it | 背景 → 挑战 → 方案 → 成果 |
   | thoughts-on-trends | 现象 → 分析 → 观点 → 展望 |
   | benchmarks | 背景 → 方法 → 数据 → 结论 |

3. **生成大纲**
   
   **开头设计：**
   - 确定 hook（问题/场景/反直觉事实）
   - 明确读者为什么要继续看
   
   **核心内容：**
   - 分 3-5 个部分
   - 每部分围绕一个要点
   - 确定每部分的素材来源
   
   **结尾设计：**
   - 与开头呼应
   - 行动号召或开放问题

4. **更新 spec.md**
   将生成的大纲写入 `spec.md` 的「文章大纲」部分

## 输出示例

```markdown
✅ 大纲生成完成

📋 写作模式：lessons-learned

## 大纲预览

### 1. 开头：共鸣痛点
- Hook：你有没有遇到过这种情况？
- 场景：每次和 AI 协作都要重新「教育」它

### 2. 问题本质
- 碎片化的 prompt → 碎片化的执行
- 案例：数据不可变原则被忽略

### 3. 解法：借鉴 SDD
- 核心理念：规范优先
- 三个关键机制

### 4. 实操步骤
- 提炼编程原则
- 结构化表达
- 建立沉淀机制

### 5. 结尾
- 少吃低级的苦
- 把精力留给更高级的挑战

📋 已更新到 spec.md

🎯 下一步：运行 `/writing.draft` 生成初稿
```

## 注意事项

- 大纲是骨架，不是最终结构
- 写作过程中可以根据需要调整
- 核心是确保每部分都服务于核心论点
