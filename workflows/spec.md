# /writing.spec

检查或完善选题规格。

## 用法

```
/writing.spec [文章目录]
```

## 执行步骤

1. **读取 spec.md**
   检查文件是否存在，读取当前内容

2. **完整性检查**
   
   | 检查项 | 状态 |
   |--------|------|
   | 一句话主题 | ✅/❌ |
   | 核心论点 | ✅/❌ |
   | 独特视角 | ✅/❌ |
   | 目标读者 | ✅/❌ |
   | 读者收获 | ✅/❌ |
   | 素材来源 | ✅/❌ |

3. **待澄清检查**
   - 列出所有 `[NEEDS CLARIFICATION]` 标记
   - 标记哪些已解决、哪些待解决

4. **引导完善**
   对于未填写的必填项，交互式引导填写

## 必填项

以下字段必须填写才能进入下一阶段：

- ✅ 一句话主题
- ✅ 核心论点
- ✅ 目标读者
- ✅ 至少 1 个读者收获

## 输出示例

### 规格完整

```markdown
✅ 规格检查通过

📋 核心定义：
- 主题：用 SDD 思想驯服 AI
- 论点：痛苦不是 AI 不强，是意图没有结构化
- 读者：使用 AI 编程的开发者

📋 待澄清问题：0 个

🎯 下一步：运行 `/writing.plan` 生成大纲
```

### 规格不完整

```markdown
⚠️ 规格检查未通过

❌ 缺失项：
- 核心论点（必填）
- 读者收获（至少 1 个）

⚠️ 待澄清问题：2 个未解决
- [ ] 技术深度：要不要深入讲细节？
- [ ] 案例来源：用真实案例还是构造场景？

请补充以上内容后再次运行 `/writing.spec`
```
