# /writing.draft

基于规格和大纲生成初稿。

## 用法

```
/writing.draft [文章目录]
```

## 前置条件

- `spec.md` 规格完整（核心定义已填写）
- 所有 `[NEEDS CLARIFICATION]` 已解决
- 建议先运行 `/writing.plan` 生成大纲

## 执行步骤

1. **验证规格**
   - 检查 `spec.md` 是否完整
   - 如不完整，提示先运行 `/writing.spec`

2. **读取素材**
   - 读取 `素材/` 目录下的所有文件
   - 提取关键信息和引用

3. **选择写作模板**
   根据 `spec.md` 中的写作模式，选择对应模板：
   - `lessons-learned` → 经验教训型
   - `bug-hunt` → 调试故事型
   - `thoughts-on-trends` → 行业趋势型
   - 其他模式...

4. **生成初稿**
   按照以下原则生成：
   
   **来自写作宪法：**
   - 真诚，不浮夸
   - 读者优先
   - 一文一点
   - 口语化表达
   - 去 AI 味
   
   **结构要求：**
   - 开头：3 行内抓住注意力
   - 核心：每部分围绕一个要点
   - 结尾：与开头呼应，有行动号召

5. **输出初稿**
   - 写入 `初稿.md`
   - 更新 `spec.md` 阶段为「初稿」

## 输出示例

```markdown
✅ 初稿生成完成

📄 文件：初稿.md
📊 字数：约 2500 字
🎯 写作模式：lessons-learned

📋 下一步：
1. 阅读初稿，标记需要修改的地方
2. 运行 `/writing.review` 进行审查
```

## 注意事项

- 初稿是草稿，不要追求完美
- 优先保证结构完整和核心论点清晰
- 细节润色留给后续阶段
