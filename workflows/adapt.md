# /writing.adapt

适配不同发布平台。

## 用法

```
/writing.adapt [文章目录] --to [平台]
```

## 支持的平台

| 平台 | 参数 | 目标字数 | 输出文件 |
|------|------|----------|----------|
| 技术博客 | `blog` | 3000+ | `blog.md` |
| 公众号 | `wechat` | ~2000 | `公众号版.md` |
| 小红书 | `rednote` | ~500 | `小红书图文/文案.txt` |

## 执行步骤

1. **读取润色版**
   读取 `润色版.md` 作为源文件

2. **根据平台调整**

### 技术博客 (blog)
- 保持完整内容
- 确保代码块格式正确
- 添加目录导航（如果超过 3000 字）
- 优化 SEO（标题、描述、标签）

### 公众号 (wechat)
- 精简到 2000 字左右
- 开头 3 行必须抓眼球
- 分段更短（适合手机阅读）
- 简化代码示例（或转为图片）
- 添加引导关注的结尾

### 小红书 (rednote)
- 极度精简到 500 字以内
- 纯文本格式（无 Markdown）
- 口语化、接地气
- 每段 2-3 行
- 底部添加标签（#AI编程 #程序员 等）
- 生成封面图（1080×1440，3:4 竖图）
  - 上半：主题配图（AI 生成或 slides 第一帧）
  - 下半：纯色底 + 大字标题（>= 72pt，高对比度，禁白底）
  - 输出到 `小红书图文/封面图.png`

3. **输出适配版本**
   - 写入对应文件
   - 更新 `spec.md` 发布计划

## 输出示例

### 公众号适配

```markdown
✅ 公众号版本生成完成

📄 文件：公众号版.md
📊 字数：约 2100 字

🔧 主要调整：
- 精简了技术细节部分
- 简化代码示例为文字描述
- 优化开头吸引力
- 添加关注引导

📋 发布清单：
- [ ] 复制到公众号编辑器
- [ ] 添加封面图
- [ ] 设置摘要
- [ ] 预览检查
- [ ] 发布
```

### 小红书适配

```markdown
✅ 小红书版本生成完成

📄 文案：小红书图文/文案.txt（约 480 字）
🎨 封面：小红书图文/封面图.png（1080×1440）

🔧 特点：
- 纯文本格式 + 口语化表达
- 3:4 竖图封面（配图 + 大字标题）
- 已添加标签

📋 发布清单：
- [ ] 检查文案和封面
- [ ] 发布（MCP 自动 / 手动上传）
```

## 注意事项

- 适配不是简单删减，而是重新组织
- 每个平台的核心论点保持一致
- 但表达方式和详略程度因平台而异
