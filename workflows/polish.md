# /writing.polish

润色初稿。

## 用法

```
/writing.polish [文章目录]
```

## 执行步骤

1. **读取输入**
   - 读取 `初稿.md`
   - 读取 `review.md`（如果存在）

2. **应用去 AI 味规则**
   
   | 原词 | 替换为 |
   |------|--------|
   | 本质上 | 说白了 / 其实 |
   | 揭示了 / 表明了 | 说明 / 证明 |
   | 这意味着什么？ | 所以呢？ |
   | 十分 / 非常 | 很 / 特别 / 删除 |
   | 护城河 / 赋能 | 删除 |

3. **优化语言表达**
   - 拆分超过 25 字的长句
   - 打散过于工整的排比
   - 减少连续设问句
   - 技术词口语化

4. **处理 review 中的问题**
   根据 `review.md` 中标记的问题逐一修改

5. **输出润色版**
   - 写入 `润色版.md`
   - 更新 `spec.md` 阶段为「润色」

## 润色原则

**保持：**
- 核心论点不变
- 文章结构不变
- 关键信息不丢失

**优化：**
- 语言更口语化
- 节奏更流畅
- 去除 AI 味

## 输出示例

```markdown
✅ 润色完成

📄 文件：润色版.md
📊 字数：约 2400 字（精简 100 字）

🔧 主要修改：
- 替换 3 处 AI 特征词
- 拆分 2 个长句
- 优化 1 处排比结构

📋 下一步：
1. 阅读润色版，确认修改
2. 运行 `/writing.adapt --to [平台]` 适配发布平台
```
