# /writing.publish

标记发布并存档。

## 用法

```
/writing.publish [文章目录] --platform [平台]
/writing.publish [文章目录] --all
```

## 参数

| 参数 | 说明 |
|------|------|
| `--platform blog` | 标记技术博客已发布 |
| `--platform wechat` | 标记公众号已发布 |
| `--platform rednote` | 标记小红书已发布 |
| `--all` | 标记所有平台已发布 |
| `--done` | 确认发布完成，更新状态 |

## 执行步骤

1. **发布前检查**
   - 确认 `spec.md` 阶段为「润色」或「待发布」
   - 确认目标平台的版本文件存在
   - 检查 `review.md` 审查结论为「通过」

2. **生成发布清单**
   
   ```markdown
   ## 发布清单
   
   ### 基本信息
   - 标题：[标题]
   - 平台：[平台]
   - 版本文件：[文件路径]
   - 发布时间：[日期]
   
   ### 发布步骤
   - [ ] 复制内容到平台编辑器
   - [ ] 检查格式
   - [ ] 上传配图
   - [ ] 设置摘要/标签
   - [ ] 预览检查
   - [ ] 发布
   
   ### 发布后
   - [ ] 24h 后记录数据
   - [ ] 7d 后更新复盘
   ```

3. **更新 spec.md**
   - 更新发布计划表中对应平台的状态
   - 记录发布日期
   - 如果所有目标平台都已发布，阶段改为「已发布」

4. **提醒复盘**
   发布后提醒填写 `spec.md` 的「复盘」部分：
   - 24h 数据
   - 7d 数据
   - 30d 数据
   - 表现分析

## 输出示例

### 标记发布

```markdown
✅ 已标记 [技术博客] 发布

📋 发布清单已生成
📅 发布日期：2026-01-06

⏰ 提醒：
- 24h 后回来填写数据
- 7d 后更新长期表现

📋 spec.md 已更新：
- 技术博客状态：✅ 已发布
- 阶段：待发布（还有 1 个平台未发布）
```

### 全部发布

```markdown
✅ 所有平台已发布

📋 发布记录：
| 平台 | 发布日期 |
|------|----------|
| 技术博客 | 2026-01-06 |
| 公众号 | 2026-01-06 |

📄 spec.md 阶段已更新为：已发布

🎉 恭喜完成这篇文章！

📋 别忘了：
- 分享到社区扩大影响
- 跟踪数据表现
- 填写复盘总结经验
```

## 复盘模板

发布后建议填写：

```markdown
## 复盘

### [平台名]
| 指标 | 24h | 7d | 30d |
|------|-----|-----|-----|
| 阅读 | 500 | 2000 | 5000 |
| 点赞 | 50 | 200 | 500 |
| 评论 | 10 | 30 | 50 |

**表现分析**: 
开头的痛点共鸣效果好，评论区互动活跃。
可复用：用问句开头 + 具体案例 的模式。
```
