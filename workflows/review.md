# /writing.review

运行审查流水线。

## 用法

```
/writing.review [文章目录]
```

## 执行步骤

1. **创建审查记录**
   从 `templates/review-checklist.md` 创建 `review.md`

2. **读取待审查文章**
   优先读取 `润色版.md`，如不存在则读取 `初稿.md`

3. **逐 Gate 检查**

### Gate 1: 规格一致性
- 对比文章内容与 `spec.md` 核心论点
- 检查目标读者定位是否偏移
- 确认所有 `[NEEDS CLARIFICATION]` 已解决

### Gate 2: 主线一致性
- 检查每个段落是否支撑核心论点
- 识别跑题或无关内容
- 验证结论与开头呼应

### Gate 3: 去 AI 味
**词汇检查：**
- 搜索「本质上」「揭示了」「表明了」等
- 搜索「十分」「非常」「极其」
- 搜索商业黑话

**结构检查：**
- 统计排比数量
- 检查设问句密度
- 检查过渡句结构

**语言检查：**
- 识别超过 25 字的长句
- 标记可口语化的技术词

### Gate 4: 事实核查
- 检查数据是否有出处
- 验证技术细节准确性
- 确认案例真实性

### Gate 5: 平台适配
根据目标平台检查：
- 博客：SEO、代码块、配图
- 公众号：标题长度、开头吸引力、分段
- 小红书：纯文本、口语化、标签

4. **生成审查结论**
   - ✅ 通过 → 可进入润色/发布
   - 🔄 需修改 → 列出问题和建议
   - ❌ 返工 → 重大问题需重写

## 输出示例

```markdown
## 审查结果

### 通过的 Gate
- ✅ Gate 1: 规格一致性
- ✅ Gate 2: 主线一致性
- ✅ Gate 4: 事实核查

### 需修改
- ⚠️ Gate 3: 去 AI 味
  - 第 3 段有「本质上」→ 建议改为「说白了」
  - 第 5 段排比过多（4 个）→ 建议减少到 3 个
  
### 结论
🔄 需微调后进入润色阶段

### 下一步
1. 修改标记的问题
2. 运行 `/writing.polish` 润色
```
